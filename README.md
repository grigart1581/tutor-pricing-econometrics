# Tutor Pricing Econometrics

Учебный эконометрический проект по анализу ценообразования на рынке репетиторских услуг на основе данных Profi.ru.

Цель проекта — оценить, какие факторы связаны с почасовой ставкой репетитора ЕГЭ: опыт, рейтинг, количество отзывов, предмет, возможность выезда к ученику и другие характеристики профиля.

## Что сделано

- собраны данные о профилях репетиторов с Profi.ru
- подготовлены JSON-файлы профилей и отзывов
- собраны аналитические таблицы по профилям, ценам, отзывам и предметам
- отобраны активные репетиторы ЕГЭ с отзывом за последние 6 месяцев
- проведён EDA: распределение цен, различия между предметами, анализ отзывов, рейтинга и опыта
- построена хедонистическая модель ценообразования с зависимой переменной `log(price)`
- проверены предпосылки модели и использованы робастные стандартные ошибки HC3

## Данные

В проекте используются данные, собранные с открытых страниц Profi.ru.

Итоговая выборка включает:

- 1 413 активных профилей репетиторов
- 2 092 наблюдения в формате «репетитор × предмет»
- 5 ЕГЭ-направлений: математика, русский язык, английский язык, физика и информатика

## Структура проекта

В репозитории находятся скрипты для сбора данных, ноутбуки с обработкой данных, EDA и эконометрическим анализом, а также отчётные материалы

- `src/` — скрипты для сбора ссылок и данных с Profi.ru
- `notebooks/` — ноутбуки для сборки чистого датасета, первичного анализа данных и эконометрического моделирования
- `data/processed/` — обработанные таблицы, использованные в анализе
- `outputs/` — таблицы и графики, полученные на этапах EDA и моделирования
- `docs/` — отчёт и презентация по проекту

## Основные результаты

В модели значимыми факторами цены оказались репутационные сигналы и характеристики профиля: количество отзывов, опыт, рейтинг и длина описания анкеты. Предметные различия между выбранными ЕГЭ-направлениями оказались статистически незначимыми на активной выборке.

## Как запустить проект

1. Установить зависимости из `requirements.txt`
2. При необходимости запустить скрипты из папки `src/` для сбора ссылок и данных
3. Запустить ноутбук `notebooks/01_build_clean_dataset.ipynb` для подготовки чистого датасета
4. Запустить `notebooks/02_eda.ipynb` для первичного анализа данных
5. Запустить `notebooks/03_econometric_modeling.ipynb` для оценки моделей и диагностики
